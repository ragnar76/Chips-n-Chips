   Eine Übersicht über verschiedene SCSI Kommandos

   Achtung: Man kann sich auf direkter Kommandoebene sehr leicht
   den Datenbestand zerstören!

   Diese Kommandos kann man mittels einem entsprechenden Programm (z.b.
   Diskus von Uwe Seimet oder SED von Claus Brod) direkt an die ange-
   schlossenen SCSI Geräte schicken und sich gegebenfalls auch den Da-
   tenbestand zerstören wenn man mit dem Format Unit Kommando rumexpe-
   rimentiert)!

   Anhang: Es gibt kein Low-Level-Format bei Festplatten, das Formatier-
           kommando bei SCSI Geräten heißt Format Unit (wer es nicht
           glauben will, soll in die SCSI Specification gucken).

   Ein paar Erklärungen zum Schluß:

   Synchronous Mode

   Beim Sychronmode werden weniger Zwischeninformationen übertragen,
   die Datenrate steigt auf Kosten der Systemauslastung.


   SCSI Linking

   Das SCSI-Gerät kann sich diverse SCSI-Befehle merken und arbeitet sie
   dann ab nachdem es den Bus wieder freigegeben hat.


   Command Queing

   Das Gerät kann die gemerkten Befehle von eben in der Reihenfolge
   ändern und so eventuell effektiver ausführen.
