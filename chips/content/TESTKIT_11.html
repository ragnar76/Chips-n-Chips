   Beim ersten Test wird in das interne Ram geschrieben, falls ein
   Fehler auftritt erscheint die Meldung EX0 bad  bit
   Überprüfe D0-D15, AS, LDS, UDS, und R/W

   Beim zweiten Test wird ein Muster in das Ram geschrieben, falls ein
   Fehler auftritt erscheint die Meldung EX1 external RAM  error
   Überprüfe A1-A11
   EX2 external RAM error Überprüfe A12-A23

   Beim dritten Test werden die Interruptleitungen überprüft, falls ein
   Fehler auftritt erscheint die Meldung EX3 INT3 error, EX4 INT5 error
   oder EX5 INT7 error.


   Fehler code's die beim Einschalten erscheinen könnten:

   I1= Ram data line is stuck. Ram fehler mit Angabe des Defekten Chips.

   Wenn auf einen Schlag 8 Ram's einer Bank als defekt gemeldet werden
   kann auch ein Bustreiber zwischen der MMU und der betreffenden Ram-
   bank defekt sein.

   Hier ein Beispiel,
   I1 RAM/data line error 0000000000000000 1111111100000000 000008
   Hier ist U33 (Mega ST) defekt (74LS373 oder 74HC373)

   I1 RAM/data line error 0000000000000000 0000000011111111 000008
   Hier ist U36 (Mega ST) defekt (74LS373 oder 74HC373)

   Es gibt pro Rechner nur jeweils 2 Chips vom Typ 74LS244 und 74LS373
   oder die entsprechende HC-Ausführung davon.
   Man kann den defekten Treiber recht einfach ermitteln, man nimmt
   einfach ein Meßgerät und prüft den Pin 2 (256Kbit*1) oder Pin 1
   (1M*1) gegen Masse. Sollte dort nur eine Spannung von 0-1 Volt an-
   liegen, den Rechner ausschalten und vom betreffenden Ram zum Bus-
   treiber durchmessen und den gefundenen Bustreiber dann gegen einen
   neuen austauschen. Das war's.

   I2= Ram disturbance. Ram fehler kann nicht genau lokalisiert werden.

   I3= Ram addressing. Siehe I2.

   I4= MMU error. Kein DTACK Signal bei Ramzugriff.(Pin 53 MMU, Pin 27
       Blitter, Pin 10 CPU und Pin 31 am GLUE).

   I5= Ram sizing error. Ramtest nicht erfolgreich, Größe kann nicht
       ermittelt wer den, wahrscheinlich MMU oder Bustreiber defekt.
