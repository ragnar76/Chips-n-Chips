   2,5 MB Ram und Testkit Rev.4.2

   Wenn die Platinenrams als Bank 0 und die Speichererweiterung als Bank
   1 addressiert sind, tritt beim Ramtest folgender Fehler auf:

   R2 0FFC0 W:00FC R:F0FC bad bits: 12 13 14 15

   EB Bus Error  Access Address: FA09D4 Program Counter:FA09D4
   EB Bus Error  Access Address: 0C0011 Program Counter:FA09CE

   Dies ist normal, weil das Testmodul nur durchgehenden Speicher testen
   kann und einen Fehler ausgibt wenn eine Bank anders bestückt ist als
   die andere.




   Fehlercodes des Testkit's:

   Ram Fehlercode:

   R0= low memory failed while setting up to run test.

   R1= failed walking 1s or 0s. An irgendeinem Ram hat eine bzw. mehrere
       Adreßleitungen keinen Kontakt

   R2= failed address (counting pattern). An irgendeinem Ram hat eine
       bzw. mehrere Adreßleitungen keinen Kontakt zum Treiber. Ist
       eventuell ein oxidierter Sockel bzw. der Massepin eines oder
       mehrerer Ramchips hat keinen 100% Kontakt.

   R3= failed 64k boundary test. Probable failure im MMU. Unterste 64
       Kbyte Ram defekt. Diesen Bereich braucht der Prozessor für interne
       Erledigungsaufgaben (unter anderem beim Testkit).

   R4= failed while displaying area tested (Video Ram). Fehler während
       des Testens des Bildschirmspeichers.

   MegaSTE
   Cache RAM= Defektes Cache RAM gefunden

   Falcon030
   DSP4= SRAM Test Timeout (Test abgebrochen)
   DSP5= SRAM Failure (Ramfehler gefunden)


   O.S.test Fehlercode:

   Bei diesem Test werden alle Byte's der Rom's gelesen und daraus die
   Checksumme ermittelt. Es können die Checksummen für folgende Länder-
   spezifischen Versionen ermittelt werden: USA/Canada, England,
   Frankreich, Deutschland und Spanien (PAL und NTSC Norm).

   Die TOS Version 1.4 ist noch nicht implementiert.
